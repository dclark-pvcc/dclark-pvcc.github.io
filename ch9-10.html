List four disadvantages of High Level Languages:
  1. The programmer must "manually" manage the movement of data items between and among memory locations and registers.
  2. The programmer must take a microscopic view of a task, breaking it down into tiny subtasks-ADD, COMPARE, INCREMENT-that direct what is going on in individual memory locations and registers.
  3. An assembly language program is machine specific.
  4. Statements are not natural-language-like.
List four expectations progammers have for writing programs in a high-level language ("Third generation" language):
  1. The programmer need not manage the details of the movement of data items within memory or pay any attention to exactly where those items are stored.
  2. The programmer can take a macroscopic view of tasks, thinking at a higher level of problem solving-The "primitive operations" used as building blocks in algorithm construction can be larger.
  3. Programs are portable rather than machine specific.
  4. Programming statements are closer to natural language and use standard mathematical notation.
Briefly describe or define:
  1. object code: Machine language instructions.
  2. source code: High-level language instructions.
  3. procedural language: Programming language in which programs consist of sequences of statements that manipulate data items.
  4. Integrated Development Environment (IDE): A software application that combines the tools needed for software development.
  5. software development life cycle: The overall sequence of steps needed to complete a large-scale software project.
  6. waterfall model of software development: The traditional model of software development that follows a series of steps.
  7. agile model of software development: An approach to software development that emphasizes a flexible and ready response to meet a shifting target.
Describe each of these components of the software development life cycle:
  1. feasibility study: Evaluates a proposed project and compares the costs and benefits of various solutions.
  2. problem specification: Involves developing a clear, concise, and unambiguous statement of the exact problem the software is to solve.
  3. program design: Plans the structure of the software to be written.
  4. algorithm selection: Once the various subtasks have been identified, algorithms must be found to carry them out.
  5. coding: The process of translating the detailed designs into computer code.
  6. debugging: The process of locating and correcting program errors.
  7. testing: The process of ensuring consistent results in an algorithm through a series of tests.
  8. verification: Used to prove that if the input data to a program satisfies certain conditions, then, after the program has been run on this data, the output data satisfies certain other conditions.
  9. documentation: All of the written and online material that makes a program understandable.

Questions & Answers
Why does the textbook refer to "Tower of Babel" regarding computer languages?
  • In the biblical story, the people all communicated with the same language. A tempting concept for coding languages at first glance.
Why isn't there just one computer programming language?
  • Each language was designed to meet the specific needs of one particular area of application.
Why didn't FORTRAN use less than ( < ) symbol when comparing two numbers?
  • The keypunches that were used to create the punched cards on which early Fortran programs were submitted to the computer had no such symbols.
What was COBOL designed for?
  • Designed to serve business needs such as managing inventories and payrolls.
What was the language, C, designed for?
  • For writing the operating system UNIX.
What is the difference between C and C++?
  • C++ adds many new features to C, giving it more sophistication and cleaner ways to do certain tasks. Anything done in C can be done in C++.
