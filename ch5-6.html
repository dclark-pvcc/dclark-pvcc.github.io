Dylan Clark
Von Neumann computer architecture, 4 major subsystems:
  1. Memory
  2. Input/Output
  3. Arithmetic/Logic Unit (ALU)
  4. Control Unit
Von Neumann computer architecture, 3 characteristics:
  1. Four major subsystems.
  2. The stored program concept.
  3. The sequential execution of instructions.
Computer Architecture Terms:
  1. RAM:(random access memory) A memory structure in which each cell has an address and it takes the same amount of time to fetch or store any cell./li>
  2. cache: A special high-speed memory unit that keeps a copy of memory cells with a high likelihood of access in the near future.
  3. Principle of locality: The high likelihood that when a program fetches a piece of data or an instruction:
    1. It will probably use it again very soon.
    2. It will probably use the "neighbors" of this item very soon.
  4. ROM:(Read-only memory) A memory structure that can only be accessed, not written into or changed.
  5. memory cell size: The number of bits in a single memory cell.
  6. memory cell address: The minimum unit of memory access.
  7. maximum memory size: The maximum amount of memory that a computer can physically hold.
  8. register: Used to implement the fetch and store operations.
  9. fetch: Load and decode the address into the MAR then copy the contents of that memory location into the MDR.
  10. store: Load the address into the MAR, load the value into the MDR, decode the address in the MAR, then store the contents of the MDR into that memory location.
  11. memory access time: The time it takes to fetch or store the contents of a single memory cell.
  12. DASD:(direct access storage device) A mass storage device in which each unit of information is associated with a unique address, but the time to access each piece of information may not be the same.
  13. Arithmetic and Logic Unit (ALU): A computer subsystem that performs such mathematical and logical operations as addition, subtraction, and comparison for equality.
  14. Control Unit: The computer subsystem that fetches and executes instructions stored in the memory of the computer.
Chapter 6: Four things the interface between the computer and the hardware does:
  1. Hides from the user the messy details of the underlying hardware.
  2. Information is presented in a simple way about the internal structure of the system.
  3. Resources available on the computer are easier to access for users.
  4. Prevents damage to hardware, programs, and data.
Responsibilities of system software:
  1. Hides details that the user does not need to see of the internal structure of the Von Neumann architecture.
  2. Presents important information in a way that is simple to understand.
  3. An efficent and simplier way for the user to access machine resources.
  4. Provides a safe environment that is secure to operate in.
Terms:
  1. System Software: A collection of computer programs that manage the resources of a computer and facilitate access to those resources.
  2. User Interface: A visual display that the operating system provides, which is in all modern operating systems.
  3. Language Services: Programs called assemblers, compilers, and interpreters, which allow users to write programs in a high-level, user-oriented language.
  4. Memory Managers: Programs that allocate memory space for programs and data and retrieve this memory space when it is no longer needed.
  5. Information Managers: Programs made to handle the organization, storage, and retrieval of information on storage devices.
  6. I/O Managers/Systems: Allow users to easily and efficiently use the many different types of input and output devices that exist on a modern computer system.
  7. Scheduler: A Program that keeps a list of programs ready to run on the processor and selects the one that will execute next. The scheduler allows users to have several different programs active at a single time.
  8. Utilities: These collections of library routines provide a wide range of useful services either to a user or to other system routines, often organized into collections called "program libraries".
  9. Assembly Language: A low-level programming language that maps 1:1 into machine language.
Functions of an Operating System:
  1. User interface management (a receptionist)
  2. Control of access to the system and to data files (a security guard)
  3. Program scheduling and activation (a dispatcher)
  4. Efficient resource allocation (an efficiency expert)
  5. Deadlock detection and error detection (a traffic officer)
